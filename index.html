# Research

Representation Learning

 

KC Machine Learning Lab

 

At the very heart of machine learning is the representation of the features from the data. Successful deep learning models in computer vision and natural language processing are built upon learning such representations not by feature engineering but by algorithmic optimization.

 

However, such models utilize the data defined on a large, dense matrix,
which is not suitable for a sparse data like a graph, or for a medium-sized feature matrix that are common in reinforcement learning.
In addition, their learned representations are often highly entangled,
which makes it difficult to semantically understand the representation space.

 

# Representation Learning 02 * For both efficiency and transferability, it is crucial to better understand the representations that deep models learn via current optimizations for network models and then to develop compact, semantic representations of features. * Better representation of states and policies in RL. * Representations of graph-learning models and computer systems using graphs. * Representations in a non-Euclidean space and its optimization.

KC Machine Learning Lab

 

Deployment of machine learning algorithms in a real-world setting has many challenges such as data efficiency and generalization.

# Graph

Graph-based Optimization

 

KC Machine Learning Lab

 

Machine Intelligence in System Design

A graph is one of the important data structures in the theory and the engineering of computer systems. Various optimizations in computer softwares are done in the domain of graphs, including compiler optimization and garbage collection.

 

Heuristic methods are frequently used in such optimizations to find the best estimate of an exact answer when it is hard to obtain in practice.

With the progress of machine learning, it is natural to think of replacing such heuristics with learned models.

 

Graph DB and ML

 

KC Machine Learning Lab

 

Machine Intelligence in System Design

Social networks and knowledge graphs are becoming ever popular, and so does
graph databases that are more natural to store such data and express queries over the data.

Optimizing graph DBs require different approach from that of traditional relational DB, because many problems on graphs are too difficult to attack using combinatorial methods and thus rely heavily on heuristics and approximations.

Machine Intelligence in System Design

KC Machine Learning Lab

 

When designing a system, it often contains heuristics that are the result of trials and errors by many experts, whose time and effort can be better spent upon researching more challenging parts of the system.

 

      Modern computer systems contain various heuristics in both hardware          and software in order to increase their computational performance.
 

      Electronic circuit design also involves various decision makings based          upon heuristics of engineers.
 

     Controller systems in various manufacturing environments are tuned by       experts' heuristics.

 

 

Machine Intelligence in System Design

KC Machine Learning Lab

 

We hope to replace the heuristics with machine learning systems
that can learn better and faster via past experience, so that every system can adapt to the environment it is deployed.


 

 

 

 

It will also free experts from designing heuristics and let them focus on formulating optimization objectives, which will benefit both the system and the experts in the long run.

Playground

KC Machine Learning Lab

 

Open Source Framework

Making libraries reusable is arguably the most important aspect of modern software design and development, and it is even more important when developing deep models because we can trust components previously built and tested, which enables us to focus on developing and debugging newer part of the research project.

 

Playground is an ML2’s effort to build such a modular platform
to design, train, and deploy deep models. As a backend itself
it will be a great help to researchers. But when equipped with convenient web-based, interactive frontend
it will make even beginners and non-experts to
compose a deep model and its data pipeline
with a few clicks and drags.
